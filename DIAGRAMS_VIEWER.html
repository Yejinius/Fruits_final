2. 크롤링 플로우 (상세)

크롤링 프로세스:

  • 6개 카테고리(A, B, C, D, E, F) 순차 처리
  • 각 카테고리별 상품 목록 조회 → 개별 상품 상세 크롤링
  • 기존 상품이 있으면 업데이트, 없으면 새로 생성 (누적+업데이트 방식)
  • 옵션: 이미지 다운로드 (--no-images 플래그로 생략 가능)
중요: DB는 삭제되지 않고 누적됩니다. 같은 article_idx는 최신 정보로 갱신됩니다.

3. Admin 테스트 플로우 (4단계)

자동화 테스트 시나리오:

  • Step 1: Admin 사이트 로그인 (REDACTED_ID / REDACTED_PW)
  • Step 2: 고객 정보 자동 등록 (EUC-KR 인코딩 필수)
  • Step 2 Result: 등록된 고객 목록 확인
  • Step 3: 주문서 작성 (카테고리/상품 자동 선택, JavaScript 트리거)
  • Step 4: 주문 목록 확인
핵심 기술: HTTP 세션 기반 (Selenium 없음), EUC-KR 인코딩, JavaScript 주입으로 Admin JS 트리거

8. 향후 개선 3: 실제 주문 연동

목표: Young Fresh Mall의 실제 주문을 Admin 사이트에 자동 등록

프로세스:

  • 고객이 Young Fresh Mall에서 주문
  • 주문 정보가 DB에 저장 (Order 테이블)
  • 백그라운드 작업이 Admin 사이트에 자동 등록
  • 등록 결과를 DB에 업데이트 (admin_order_id, status)
